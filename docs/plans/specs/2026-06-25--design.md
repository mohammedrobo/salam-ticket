# Driver Management Dashboard — Design Spec

**Date:** 2026-06-25
**Goal:** Manage drivers on-site — scan QR to check in, manager removes when done
**Tech Stack:** Next.js 14 (App Router), SQLite (better-sqlite3), Tailwind CSS, QRCode library

---

## Overview

A simple driver queue system:
1. Manager opens dashboard → sees QR code + list of waiting drivers
2. Driver scans QR → enters name → appears on dashboard
3. Manager clicks "Done" → driver removed from queue

## User Roles

- **Manager:** Views dashboard, deletes drivers when done
- **Driver:** Scans QR, enters name once

## Pages

### `/` — Manager Dashboard
- Shows QR code (regeneratable)
- Auto-refreshing list of drivers (polling every 3s)
- Each driver row: Name, Time Scanned, Status
- "Done" button per driver to remove them

### `/scan` — Driver Check-in
- Simple form: Name input + Submit button
- On submit: saves driver to DB, shows confirmation
- Name remembered on device (localStorage)

## Data Model

```sql
CREATE TABLE drivers (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  scanned_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  status TEXT DEFAULT 'waiting' -- 'waiting' or 'done'
);
```

## API Routes

- `GET /api/drivers` — List all waiting drivers
- `POST /api/drivers` — Add new driver (name)
- `DELETE /api/drivers?id=X` — Remove driver

## QR Code
- Generated server-side as SVG
- Links to `/scan` page
- Manager can regenerate with new UUID

## Design Requirements
- Dark theme, clean and modern
- Mobile-responsive
- Large driver names for visibility from distance
